How did you interpret the user’s needs and implement them into your system design? Why is it so important to consider the user’s needs when designing?
- Designing is all about understanding the client's needs and how you could turn that into a working system based on your understanding and vision.
- I implemented my system design using keyword interpretation technique. I take important keywords and used that in how I create systems and subsystems of the DriverPass webapp.
- For example, when customer says they want users to LOGIN to the system online, then you interpret that as user wants a webpage of the system to login and access the content.

How do you approach designing software? What techniques or strategies would you use in the future to analyze and design a system?
- For me, software design is a meticulous journey. My strategy would be all about precision and active communication between client and the project team.
- Designing of a software depends entirely on how your client/company envisions the system and how you can translate that into code and app behavior.
- Having a great understanding of what the system should do and how it should do it is your bread and butter.
